2020/10/28
簡易悪堕ち処理の中身はほぼ既存の悪堕ちです。
内部的には生成時点で名前(と一部パラメータ)を変えただけの通常エネミーです。
既存の悪堕ちエネミー(ラミア・スライム・アルラウネ・触手姫・通常悪堕ち)とは異なり、
専用のエネミー番号／技を持っていません(既存エネミーに間借りしています)。
　(例示)
	早苗さんがビオラキメラを出産する場合、「早苗専用ビオラキメラ」の名を持つ「ビオラキメラ」が迷宮に生成されています。
	咲夜さんがラミアにトドメを刺された場合、「ラミア咲夜」の名を持つ「ラミア化エネミー」が迷宮に生成されています。
　New!　妖夢さんがマミーにトドメを刺された場合、「マミー妖夢」の名を持つ「マミー」が迷宮に生成されるようになります。

　◆個別エネミーの設定ファイル(敵詳細データ内)は、第２引数に
　　①悪堕ち元のキャラ　または　②母親の番号　を取る場合がありますよね。
　　前者はアルラウネ化キャラ、後者はビオラキメラや〇〇の娘。
　◆①／②どちらの意図で第２引数が渡されたのか、設定ファイル側からは判別できません。
　　(従来は寄生出産で生成されるエネミーと悪堕ち先エネミーがはっきり分かれているため、これは問題になりません。)
　　しかし簡易悪堕ちの場合、通常の出産で生成され得るエネミーのデータを使って悪堕ちします。
　　たとえば引数付きでサキュバスが生成される場合、システム側で両者の区別を付ける必要があります。
　◆RETIRE.erbの悪堕ち処理に追記。
　　エネミー生成直前、FLAG:2000を1に。
　　簡易悪堕ち対応エネミーの側では「FLAG:2000==1なら(出産ではなく)悪堕ちによる生成」と見なす。
　　これで出産経由での生成と悪堕ち経由での生成に両対応になりました。
